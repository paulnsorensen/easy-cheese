# ADR: shared docs live in the cheese skill's references dir, cited sibling-relative

### ADR-001: `skills/cheese/references/` is the home for cross-skill shared docs  [status: accepted]

- **Context:** Both install paths (`npx skills add` — SKILL.md-dir discovery; `scripts/install.sh` — `contents/skills` fetch) drop repo-root `shared/`, so every installed skill carried dangling references to handoff-gate.md, formatting.md, harness-portability.md, optional-plugins.md, semantic-backends.md, skill-authoring.md. The Agent Skills spec has no cross-skill sharing primitive; the community norm is duplication (research: `.cheese/research/shared-skill-content/`).
- **Decision:** Move the six shared docs into `skills/cheese/references/` (cheese = the always-installed entry-point router, present in `EC_FALLBACK_SKILLS`). Every reference — markdown link *and* prose — uses the relative-from-current-file form (`../cheese/references/<f>.md` from a SKILL.md; `../../cheese/references/<f>.md` from a references file), which resolves identically in the repo tree and every sibling-installed tree.
- **Alternatives:** (a) Build-step inlining per skill (the `.pyz` pattern for markdown) — rejected: N skills load N byte-identical copies into context; a single path is read once per session. (b) New dedicated `shared`/`rind` skill — works, but adds a skill-list entry and a second always-install dependency; cheese already is one. (c) Plugin packaging (`.claude-plugin/`) — preserves top-level dirs but abandons `npx skills` compatibility and rests on an unverified `${CLAUDE_PLUGIN_ROOT}`-in-SKILL.md mechanism (anthropics/claude-code#9354). (d) Patch installers — impossible for the upstream vercel-labs CLI.
- **Consequences:** Zero installer/build changes; shared docs enter the `skills/**/*.md` lint glob (closing the unlinted-`shared/` hole); `gen_docs.py`'s dedicated shared section retires. Cost: a `--skill <subset>` install without `cheese` still dangles (accepted, documented); repo-root `shared/` keeps only `scripts/`.
