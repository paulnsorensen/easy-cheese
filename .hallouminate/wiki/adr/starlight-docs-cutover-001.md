# ADR: Starlight docs cutover

The docs migration should be a clean cutover from MkDocs Material to Astro Starlight, not a compatibility layer.

## Context

The current site is built by MkDocs Material from `mkdocs.yml`, `docs/requirements.txt`, and `scripts/gen_docs.py`. The generator keeps documentation single-sourced by emitting skill pages, reference pages, install docs, shared contracts, root passthrough pages, and navigation at build time. The proposed Starlight migration keeps that generated-docs contract but changes the output backend to physical Starlight content under `src/content/docs/**` plus generated sidebar data.

## Decision

Use pnpm for the Astro/Starlight toolchain. Keep generated docs and `src/sidebar.mjs` uncommitted, regenerated by local docs commands and CI before `astro build`. Preserve rendered Mermaid diagrams through a Starlight-compatible Mermaid integration rather than degrading diagrams to plain code blocks.

## Rationale

A compatibility layer would leave two documentation contracts alive and make stale output harder to detect. Physical generated Starlight files match Astro's content pipeline while preserving the repo's existing single-source generator model. pnpm was selected explicitly during the mold gate. Mermaid rendering is user-visible on the homepage pipeline diagram and part of the current documentation experience.

## Consequences

- `scripts/gen_docs.py` becomes the migration center: it must emit Starlight frontmatter, convert MkDocs-specific generated admonitions, preserve link rewriting, and write `src/sidebar.mjs`.
- CI changes from Python/MkDocs installation plus `mkdocs build --strict` to pnpm dependency installation plus `pnpm run docs:build`.
- `docs/stylesheets/cheese.css` cannot be ported selector-for-selector because it targets MkDocs Material DOM classes; the Starlight theme CSS must preserve design intent through Starlight-compatible selectors and variables.
- Direct `astro build` is unsupported until generation has run; supported commands must run docs generation first.
